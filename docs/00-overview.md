# 项目总览

## 一句话说明

建设一个实验室局域网服务器状态监控平台，让成员通过网页查看每台服务器当前是否在线、谁在线、哪些进程占用了 CPU/GPU、各 GPU 卡的显存和算力占用情况。

## 已知服务器环境

| 编号 | 系统 | 硬件重点 | 采集重点 |
|---|---|---|---|
| 1 | Linux | 8 张 NVIDIA A100 | 用户会话、进程、CPU/内存、GPU 利用率、GPU 进程归属 |
| 2 | Windows | CPU 主机，普通显卡 | 用户会话、进程、CPU/内存，GPU 信息作为弱需求 |
| 3 | Windows | 8 张 NVIDIA P100 | 用户会话、进程、CPU/内存、GPU 利用率、GPU 进程归属 |

## 核心用户

- 实验室普通成员：查看哪台服务器空闲、谁在占用资源、当前是否适合提交任务。
- 实验室管理员：添加服务器、查看 Agent 状态、排查异常、后续可以扩展告警和资源统计。

## 产品边界

MVP 阶段只做“看清楚当前情况”，不做远程杀进程、自动调度任务、权限隔离计费，也不替代 Prometheus/Grafana 这类专业监控平台。

## 推荐方案

采用轻量 Agent 方案：

```text
Linux / Windows 服务器
  -> 本机轻量 Agent 采集资源
  -> HTTPS/HTTP 上报 Spring Boot 后端
  -> 后端存储最新快照和短期历史
  -> Vue 页面展示看板、服务器列表、进程列表、GPU 详情
```

这样做的原因：

- Linux 和 Windows 采集方式差异很大，本机 Agent 最容易保证准确性。
- 后端不需要保存每台服务器的管理员密码，安全边界更清楚。
- 后续新增服务器时，只要在网页添加服务器并安装 Agent 即可。
- Agent 可以独立升级，前后端不需要关心每种系统命令细节。
